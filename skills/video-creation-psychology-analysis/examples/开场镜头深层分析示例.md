# 示例：开场镜头深层分析

## 镜头：00:00｜电脑套壳 + B站播放页 + 缓慢推入

**口播/文案要点：**  
“从发布第一个视频开始到现在……”

**画面/素材：**  
B站视频播放页被放进电脑屏幕套壳中，画面居中，黑底留白，整体像一个被摆到观众面前的创作证据。

**动效/剪辑：**  
电脑框定场，页面轻微推入或稳定停留，节奏不快，强调进入感而非信息轰炸。

**表层作用：**  
告诉观众：这里讲的是作者真实发布过的视频和创作经历。

**深层心理/导演想法：**  
这句话的核心不是“我有一个视频”，而是“我有一段真实的创作路径”。观众在视频开头最关心的不是细节，而是：这个人凭什么讲视频制作？所以创作者需要先交出一个证据，但不能一上来堆太多信息。电脑套壳相当于一个“创作现场入口”，让观众感觉自己正在进入作者的工作台。B站页面相当于“真实作品证据”，让经验分享有落点。缓慢推入则让开场有仪式感，避免像普通录屏教程一样廉价。

**为什么偏偏这样配：**  
这个组合同时解决了三件事：

1. 建立真实感：B站页面说明作品真实存在；
2. 建立场景感：电脑套壳让页面不只是截图，而是一个被观看的工作现场；
3. 建立叙事感：推入动效让观众从外部慢慢进入作者经历。

**为什么不用其他方案：**

- 如果用纯录屏：会太像操作教程，观众会期待你教软件，而不是听创作复盘。
- 如果用作品截图墙：信息密度太高，容易变成“我做了很多作品”的炫耀感。
- 如果用纯文字：缺少证据感，观众不会立刻相信这是来自真实创作经验。
- 如果用真人出镜：注意力会转移到人身上，而不是作品和创作过程。
- 如果用论文/资料堆：资料感太重，会提前进入知识研究段落，不适合开场。

**可复用思路：**  
当你的口播是“我从什么时候开始做某件事”“我做过某个项目”“我长期积累了某个经验”时，可以用：

```text
真实作品页面 / 后台 / 项目页
+ 一个场景容器，例如电脑、手机、桌面、档案夹
+ 缓慢推入或稳定停留
```

目的不是炫耀成果，而是建立“我真的做过，所以我接下来讲的方法有来源”。
